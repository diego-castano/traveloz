- ordenServicios: empty array (will be managed by the app)

**Junction/Assignment arrays:**

`SEED_PAQUETE_AEREOS: PaqueteAereo[]`:
- Each paquete gets 1 aereo assignment (the flight for that destination)
- aereoId must reference a valid aereo-N from aereos.ts
- textoDisplay: null (auto-generated from aereo data)
- orden: 1

`SEED_PAQUETE_ALOJAMIENTOS: PaqueteAlojamiento[]`:
- Each paquete gets 1-2 hotel assignments
- paquete-6 "Rio + Buzios" gets 2 hotels (multi-destino) with nochesEnEste split (e.g., 5 + 5)
- alojamientoId must reference valid alojamiento-N from alojamientos.ts

`SEED_PAQUETE_TRASLADOS: PaqueteTraslado[]`:
- Most paquetes get 1-2 traslado assignments (airport transfers)
- trasladoId must reference valid traslado-N from traslados.ts

`SEED_PAQUETE_SEGUROS: PaqueteSeguro[]`:
- Each paquete gets 1 seguro assignment
- seguroId must reference valid seguro-N from seguros.ts
- diasCobertura: same as paquete.noches or null

`SEED_PAQUETE_CIRCUITOS: PaqueteCircuito[]`:
- Only paquete-5 "Portugal Magnifico" and paquete-14 "Turquia Magica" get circuito assignments
- circuitoId: circuito-1 and circuito-2 respectively

`SEED_PAQUETE_FOTOS: PaqueteFoto[]`:
- 2-3 photos per paquete using Unsplash placeholder URLs
- Use real Unsplash photo IDs for travel-relevant imagery

`SEED_PAQUETE_ETIQUETAS: PaqueteEtiqueta[]`:
- Each paquete gets 1-3 etiqueta assignments matching its destination/type
- etiquetaId must reference valid etiqueta-N from catalogos.ts

**Pricing computation for each paquete:**
- Look at the assigned services and their prices from the sibling seed files
- netoCalculado = sum of (aereo adult price for matching periodo + hotel per-night * noches + traslado prices + seguro per-day * noches + circuito price if any)
- markup: 25-40% (vary across paquetes)
- precioVenta = Math.round(netoCalculado * (1 + markup/100))
- These are pre-computed static values in the seed data, not computed at runtime

**Cross-reference validation:**
- Every aereoId, alojamientoId, trasladoId, seguroId, circuitoId, etiquetaId, temporadaId, tipoPaqueteId must reference actual IDs from the sibling seed data files
- Comment each assignment with the entity name for clarity (e.g., `aereoId: 'aereo-1', // MVD->GIG->MVD Azul`)
