Create CatalogProvider.tsx following the split state/dispatch context pattern from the research. This provider manages all reference/catalog data: Temporada, TipoPaquete, Etiqueta, Pais, Ciudad, Regimen, and Proveedor.

**File structure:**

1. **"use client"** directive at top

2. **Imports:**
   - React: createContext, useContext, useReducer, useMemo, Dispatch
   - Types from @/lib/types: Temporada, TipoPaquete, Etiqueta, Pais, Ciudad, Regimen, Proveedor
   - Seed data from @/lib/data: SEED_TEMPORADAS, SEED_TIPOS_PAQUETE, SEED_ETIQUETAS, SEED_PAISES, SEED_CIUDADES, SEED_REGIMENES, SEED_PROVEEDORES
   - useBrand from ./BrandProvider

3. **State interface:**
```typescript
interface CatalogState {
  temporadas: Temporada[];
  tiposPaquete: TipoPaquete[];
  etiquetas: Etiqueta[];
  paises: Pais[];
  ciudades: Ciudad[];
  regimenes: Regimen[];
  proveedores: Proveedor[];
}
```

4. **Action type** (discriminated union):
```typescript
type CatalogAction =
  | { type: 'ADD_TEMPORADA'; payload: Temporada }
  | { type: 'UPDATE_TEMPORADA'; payload: Temporada }
  | { type: 'DELETE_TEMPORADA'; payload: string }
  // ... same pattern for TipoPaquete, Etiqueta, Pais, Ciudad, Regimen, Proveedor
  // DELETE for Proveedor uses soft delete (sets deletedAt)
  // DELETE for catalog entities (Temporada, Tipo, Etiqueta, Regimen) is hard delete (filter out)
  // Pais and Ciudad: hard delete
```

5. **Reducer function:**
- Switch on action.type
- ADD_*: return [...state.entities, action.payload]
- UPDATE_*: return state.entities.map(e => e.id === action.payload.id ? action.payload : e)
- DELETE_TEMPORADA/TIPO/ETIQUETA/REGIMEN/PAIS/CIUDAD: filter out by id (hard delete for reference data)
- DELETE_PROVEEDOR: set deletedAt to new Date().toISOString() (soft delete)
- Always return new state object (immutable updates)

6. **Two separate contexts:**
```typescript
const CatalogStateContext = createContext<CatalogState | null>(null);
const CatalogDispatchContext = createContext<Dispatch<CatalogAction> | null>(null);
```

7. **Provider component:**
```typescript
export function CatalogProvider({ children }: { children: React.ReactNode }) {
  const [state, dispatch] = useReducer(catalogReducer, initialState);
  return (
    <CatalogStateContext.Provider value={state}>
      <CatalogDispatchContext.Provider value={dispatch}>
        {children}
      </CatalogDispatchContext.Provider>
    </CatalogStateContext.Provider>
  );
}
```
initialState loaded from SEED_* imports.

8. **Raw state/dispatch hooks:**
```typescript
export function useCatalogState(): CatalogState { ... throw if null }
export function useCatalogDispatch(): Dispatch<CatalogAction> { ... throw if null }
```

9. **Brand-filtered selector hooks** (each uses useBrand() and useMemo):
```typescript
export function useTemporadas(): Temporada[] {
  const { activeBrandId } = useBrand();
  const state = useCatalogState();
  return useMemo(() => state.temporadas.filter(t => t.brandId === activeBrandId), [state.temporadas, activeBrandId]);
}
```
Same pattern for: useTiposPaquete, useEtiquetas, useRegimenes, useProveedores (proveedores also filter !deletedAt).
- usePaises: filter by brandId, and for each pais, attach its ciudades from state.ciudades where paisId matches.
  Return type: `(Pais & { ciudades: Ciudad[] })[]`

10. **CRUD action hook:**
```typescript
export function useCatalogActions() {
  const dispatch = useCatalogDispatch();
  return useMemo(() => ({
    createTemporada: (data: Omit<Temporada, 'id' | 'createdAt' | 'updatedAt'>) => {
      const entity: Temporada = { ...data, id: crypto.randomUUID(), createdAt: new Date().toISOString(), updatedAt: new Date().toISOString() };
      dispatch({ type: 'ADD_TEMPORADA', payload: entity });
      return entity;
    },
    updateTemporada: (entity: Temporada) => dispatch({ type: 'UPDATE_TEMPORADA', payload: { ...entity, updatedAt: new Date().toISOString() } }),
    deleteTemporada: (id: string) => dispatch({ type: 'DELETE_TEMPORADA', payload: id }),
    // ... same for all 7 entity types
  }), [dispatch]);
}
```

**Important patterns to follow:**
- useMemo on selector results (dependency on specific state slice + activeBrandId)
- Split state/dispatch contexts (as specified in research Pattern 3)
- Discriminated union actions (research Pattern 4)
- Error messages: "useXxx must be used within a <CatalogProvider>"
- Do NOT use `any` anywhere
