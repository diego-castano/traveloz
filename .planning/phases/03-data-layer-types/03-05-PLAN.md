Create PackageProvider.tsx following the same split state/dispatch pattern as CatalogProvider and ServiceProvider. This is the most complex provider because it manages the central Paquete entity plus all 7 junction/sub-entity types.

**State interface:**
```typescript
interface PackageState {
  paquetes: Paquete[];
  paqueteAereos: PaqueteAereo[];
  paqueteAlojamientos: PaqueteAlojamiento[];
  paqueteTraslados: PaqueteTraslado[];
  paqueteSeguros: PaqueteSeguro[];
  paqueteCircuitos: PaqueteCircuito[];
  paqueteFotos: PaqueteFoto[];
  paqueteEtiquetas: PaqueteEtiqueta[];
}
```

**Action type** (discriminated union):
- ADD/UPDATE/DELETE for Paquete (soft delete)
- ADD/DELETE for each junction type: PaqueteAereo, PaqueteAlojamiento, PaqueteTraslado, PaqueteSeguro, PaqueteCircuito, PaqueteFoto, PaqueteEtiqueta (hard delete -- filter out)
- UPDATE for PaqueteAlojamiento (to change nochesEnEste), PaqueteFoto (to change orden), other junctions (to change textoDisplay/orden)
- CLONE_PAQUETE: creates copy with new ID, "Copia de {titulo}", estado BORRADOR, plus copies all junction records with new IDs

**Reducer:** Immutable updates for all action types.

CLONE_PAQUETE special logic in reducer:
```typescript
case 'CLONE_PAQUETE': {
  const source = state.paquetes.find(p => p.id === action.payload);
  if (!source) return state;
  const newId = crypto.randomUUID();
  const now = new Date().toISOString();
  const cloned: Paquete = {
    ...source,
    id: newId,
    titulo: `Copia de ${source.titulo}`,
    estado: 'BORRADOR',
    destacado: false,
    createdAt: now,
    updatedAt: now,
    deletedAt: null,
  };
  // Clone all junction records
  const clonedAereos = state.paqueteAereos
    .filter(pa => pa.paqueteId === source.id)
    .map(pa => ({ ...pa, id: crypto.randomUUID(), paqueteId: newId }));
  // ... same for alojamientos, traslados, seguros, circuitos, fotos, etiquetas
  return {
    ...state,
    paquetes: [...state.paquetes, cloned],
    paqueteAereos: [...state.paqueteAereos, ...clonedAereos],
    // ... etc
  };
}
```

**Split contexts:** PackageStateContext + PackageDispatchContext

**Provider component:** Initialize state from SEED_PAQUETES and all SEED_PAQUETE_* imports.

**Brand-filtered selector hooks:**
- `usePaquetes(): Paquete[]` -- filter by activeBrandId AND !deletedAt, memoized
- `usePaqueteById(id: string): Paquete | undefined` -- find by id in state.paquetes (no brand filter needed since ID is unique), check !deletedAt

**CRUD + clone + assign action hook:**
```typescript
export function usePackageActions() {
  const dispatch = usePackageDispatch();
  return useMemo(() => ({
    // Paquete CRUD
    createPaquete: (data: Omit<Paquete, 'id' | 'createdAt' | 'updatedAt' | 'deletedAt'>) => {
      const paquete: Paquete = { ...data, id: crypto.randomUUID(), createdAt: new Date().toISOString(), updatedAt: new Date().toISOString(), deletedAt: null };
      dispatch({ type: 'ADD_PAQUETE', payload: paquete });
      return paquete;
    },
    updatePaquete: (paquete: Paquete) => dispatch({ type: 'UPDATE_PAQUETE', payload: { ...paquete, updatedAt: new Date().toISOString() } }),
    deletePaquete: (id: string) => dispatch({ type: 'DELETE_PAQUETE', payload: id }),
    clonePaquete: (id: string) => dispatch({ type: 'CLONE_PAQUETE', payload: id }),

    // Service assignment helpers
    assignAereo: (data: Omit<PaqueteAereo, 'id'>) => {
      const assignment: PaqueteAereo = { ...data, id: crypto.randomUUID() };
      dispatch({ type: 'ADD_PAQUETE_AEREO', payload: assignment });
      return assignment;
    },
    removeAereo: (id: string) => dispatch({ type: 'DELETE_PAQUETE_AEREO', payload: id }),
    updateAereoAssignment: (assignment: PaqueteAereo) => dispatch({ type: 'UPDATE_PAQUETE_AEREO', payload: assignment }),

    // Same pattern for: assignAlojamiento/removeAlojamiento/updateAlojamientoAssignment,
    // assignTraslado/removeTraslado/updateTrasladoAssignment,
    // assignSeguro/removeSeguro/updateSeguroAssignment,
    // assignCircuito/removeCircuito/updateCircuitoAssignment

    // Photo management
    addFoto: (data: Omit<PaqueteFoto, 'id'>) => {
      const foto: PaqueteFoto = { ...data, id: crypto.randomUUID() };
      dispatch({ type: 'ADD_PAQUETE_FOTO', payload: foto });
      return foto;
    },
    removeFoto: (id: string) => dispatch({ type: 'DELETE_PAQUETE_FOTO', payload: id }),
    updateFoto: (foto: PaqueteFoto) => dispatch({ type: 'UPDATE_PAQUETE_FOTO', payload: foto }),

    // Etiqueta assignment
    assignEtiqueta: (data: Omit<PaqueteEtiqueta, 'id'>) => {
      const assignment: PaqueteEtiqueta = { ...data, id: crypto.randomUUID() };
      dispatch({ type: 'ADD_PAQUETE_ETIQUETA', payload: assignment });
      return assignment;
    },
    removeEtiqueta: (id: string) => dispatch({ type: 'DELETE_PAQUETE_ETIQUETA', payload: id }),
  }), [dispatch]);
}
```

**Helper hooks for accessing junction data:**

```typescript
// Get all service assignments for a specific paquete
export function usePaqueteServices(paqueteId: string) {
  const state = usePackageState();
  return useMemo(() => ({
    aereos: state.paqueteAereos.filter(pa => pa.paqueteId === paqueteId),
    alojamientos: state.paqueteAlojamientos.filter(pa => pa.paqueteId === paqueteId),
    traslados: state.paqueteTraslados.filter(pt => pt.paqueteId === paqueteId),
    seguros: state.paqueteSeguros.filter(ps => ps.paqueteId === paqueteId),
    circuitos: state.paqueteCircuitos.filter(pc => pc.paqueteId === paqueteId),
    fotos: state.paqueteFotos.filter(pf => pf.paqueteId === paqueteId).sort((a, b) => a.orden - b.orden),
    etiquetas: state.paqueteEtiquetas.filter(pe => pe.paqueteId === paqueteId),
  }), [state, paqueteId]);
}
```

Update the existing Providers.tsx to include the 3 new providers in the correct nesting order. The order matters because React Context flows downward -- a provider can only consume contexts mounted ABOVE it.

**Required nesting order (outermost to innermost):**
```
AuthProvider > BrandProvider > CatalogProvider > ServiceProvider > PackageProvider > ToastProvider
```

Rationale:
- AuthProvider: no deps (outermost)
- BrandProvider: reads AuthProvider (for user's brand)
- CatalogProvider: reads BrandProvider (for activeBrandId filtering)
- ServiceProvider: reads BrandProvider (for activeBrandId filtering)
- PackageProvider: reads BrandProvider (for activeBrandId filtering)
- ToastProvider: reads nothing (innermost, just provides toast UI)

**Updated file:**

```typescript
    <AuthProvider>

            <PackageProvider>
              <ToastProvider>{children}</ToastProvider>
            </PackageProvider>
          </ServiceProvider>

}
```
