  <files>src/app/(admin)/paquetes/[slug]/_components/ServiciosTab.tsx</files>
  <action>
Create the Servicios tab component showing all assigned services grouped by type.

1. **Props:** `{ paquete: Paquete }`.

2. **Hooks:**
   - `usePaqueteServices(paquete.id)` -- returns `{ aereos, alojamientos, traslados, seguros, circuitos, fotos, etiquetas }` (junction records).
   - `useAereos()`, `useAlojamientos()`, `useTraslados()`, `useSeguros()`, `useCircuitos()` from ServiceProvider -- to look up service names/details from junction foreign keys.
   - `usePackageActions()` for `removeAereo`, `removeAlojamiento`, `removeTraslado`, `removeSeguro`, `removeCircuito`.
   - `useAuth()` for `canEdit`.
   - `useState` for `modalOpen: boolean`.

3. **Service group sections:** For each service type, render a section with:
   - Section header with service type name (e.g., "Aereos", "Alojamientos") and a count badge.
   - A list of assigned items in a glass Card. Each item shows:
     - For aereos: `aereo.ruta` + `aereo.destino` + `aereo.aerolinea`
     - For alojamientos: `alojamiento.nombre` + star rating + ciudad (need to look up ciudad name)
     - For traslados: `traslado.nombre` + `traslado.tipo`
     - For seguros: `seguro.plan` + `seguro.proveedor` (look up from proveedores)
     - For circuitos: `circuito.nombre`
   - Each item has a remove button (X icon) if `canEdit`. Clicking calls the appropriate `removeXxx(assignment.id)`.

4. **Reorder via HTML5 drag-and-drop:** Each service item is a `<div draggable={canEdit}` with `onDragStart`, `onDragOver`, `onDrop` handlers. Reorder updates the paquete's `ordenServicios` array via `updatePaquete`. The `ordenServicios` field on Paquete stores an ordered list of assignment IDs. Use a local state for drag source index and swap on drop. This is a simplified reorder -- no animation needed, just functional.

5. **"Agregar Servicio" button:** At the top of the tab, conditionally rendered if `canEdit`. Opens the `ServiceSelectorModal`.

6. **Empty state:** If no services assigned, show "No hay servicios asignados" with a call-to-action button.

7. **All text in Spanish:** "Servicios Asignados", "Agregar Servicio", "No hay servicios asignados", "Aereos", "Alojamientos", "Traslados", "Seguros", "Circuitos".
  </action>

**ServiceSelectorModal.tsx:**

1. **Props:** `{ paqueteId: string; open: boolean; onOpenChange: (open: boolean) => void }`.

2. **Structure:** `Modal` > `ModalHeader("Agregar Servicio")` > `ModalBody` containing `Tabs` with **`layoutId="serviceSelectorTabs"`** (CRITICAL: must differ from the parent page tabs which use default `layoutId="activeTab"`, otherwise the animated underline breaks).

3. **5 TabsTrigger entries:** "Aereos", "Alojamientos", "Traslados", "Seguros", "Circuitos".

4. **Each TabsContent:** Shows a list of available services (from ServiceProvider hooks) that are NOT already assigned to this paquete. Filter logic:
   - Get assigned IDs from `usePaqueteServices(paqueteId)` -- e.g., `services.aereos.map(a => a.aereoId)`.
   - Get all services from `useAereos()`.
   - Filter: `aereos.filter(a => !assignedAereoIds.includes(a.id))`.

5. **Each available service row:** Rendered in a simple table or list inside the Card:
   - Aereos: ruta, destino, aerolinea -- "Agregar" button
   - Alojamientos: nombre, categoria stars, ciudad -- "Agregar" button
   - Traslados: nombre, tipo, precio -- "Agregar" button
   - Seguros: plan, cobertura, costoPorDia -- "Agregar" button
   - Circuitos: nombre -- "Agregar" button

6. **"Agregar" button handler:** Call the appropriate `assignXxx` from `usePackageActions()`. For example:
   - `assignAereo({ paqueteId, aereoId: aereo.id, precioAereoId: null, notas: '' })` -- the precioAereoId can be set later in Precios tab.

   - `assignTraslado({ paqueteId, trasladoId: traslado.id, notas: '' })`.
   - `assignSeguro({ paqueteId, seguroId: seguro.id, diasCobertura: null, notas: '' })`.
   - `assignCircuito({ paqueteId, circuitoId: circuito.id, precioCircuitoId: null, notas: '' })`.
   After assigning, show a success toast and the item disappears from the available list (since state updates trigger re-render).

7. **Empty state per tab:** If all services of a type are already assigned, show "Todos los {tipo} estan asignados".

8. **Modal size:** `size="lg"` for enough space for the tabbed content.
