**Create `src/components/providers/UserProvider.tsx`** following the exact CatalogProvider pattern (split state/dispatch contexts, useReducer, action hooks with useMemo):

```
"use client";
import { createContext, useContext, useReducer, useMemo, type Dispatch } from "react";
import type { AuthUser, Role } from "@/lib/auth";
import { DEMO_USERS } from "@/lib/auth";
```

**Action types (discriminated union):**
- `{ type: "ADD_USER"; payload: AuthUser }`
- `{ type: "UPDATE_USER"; payload: AuthUser }`
- `{ type: "DELETE_USER"; payload: string }` (hard delete — no deletedAt on AuthUser)

**Reducer:** ADD_USER appends, UPDATE_USER maps and replaces by id, DELETE_USER filters out by id.

**Initial state:** `DEMO_USERS` from `src/lib/auth.ts` (5 demo users).

**Contexts:** Split `UserStateContext` (AuthUser[]) and `UserDispatchContext` (Dispatch).

**Provider component:** `export function UserProvider({ children })` — useReducer with userReducer and DEMO_USERS initial state.

**Hooks:**
- `export function useUsers(): AuthUser[]` — returns full user list (NOT brand-filtered — ADMIN sees all users across all brands)
- `export function useUserActions()` — returns memoized object:
  - `createUser(data: Omit<AuthUser, "id">): AuthUser` — generates id via `crypto.randomUUID()`, dispatches ADD_USER, returns new user
  - `updateUser(user: AuthUser): void` — dispatches UPDATE_USER
  - `deleteUser(id: string): void` — dispatches DELETE_USER

Both hooks throw descriptive errors if used outside provider.

```typescript
const roleBadgeVariant: Record<Role, "active" | "pending" | "draft"> = {
  ADMIN: "active",
  VENDEDOR: "pending",
  MARKETING: "draft",
};

  ADMIN: "Admin",
  VENDEDOR: "Vendedor",
  MARKETING: "Marketing",
};
```
