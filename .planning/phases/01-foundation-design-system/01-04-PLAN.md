  <name>Task 1: Build Modal with Radix Dialog + Motion AnimatePresence + liquid glass</name>
  <files>src/components/ui/Modal.tsx</files>
  <action>
Create Modal using Radix Dialog + Motion AnimatePresence. Follow the EXACT pattern from research to avoid the pointer-events bug.

1. **Imports**: `import { Dialog } from 'radix-ui'`, `import { AnimatePresence, motion } from 'motion/react'`, glassMaterials, interactions, springs from lib.

2. **Modal component** (main wrapper):
   - Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `children: React.ReactNode`, `size?: 'sm' | 'md' | 'lg' | 'xl' | 'full'`
   - Size mapping: sm=420px, md=560px, lg=720px, xl=900px, full=calc(100vw - 80px)
   - Structure:
     ```
     <Dialog.Root open={open} onOpenChange={onOpenChange}>
       <AnimatePresence>
         {open && (
           <Dialog.Portal forceMount>
             <Dialog.Overlay forceMount asChild>
               <motion.div overlay />
             </Dialog.Overlay>
             <Dialog.Content forceMount asChild>
               <motion.div content>{children}</motion.div>
             </Dialog.Content>
           </Dialog.Portal>
         )}
       </AnimatePresence>
     </Dialog.Root>
     ```
   - CRITICAL: `forceMount` on BOTH Portal, Overlay, AND Content. This prevents the pointer-events bug.

3. **Overlay animation**:
   - className: `fixed inset-0 z-[30]`
   - style: `{ background: 'rgba(26,26,46,0.45)' }`
   - initial: `{ opacity: 0 }`
   - animate: `{ opacity: 1, backdropFilter: 'blur(8px)' }`
   - exit: `{ opacity: 0, backdropFilter: 'blur(0px)' }`
   - transition: `{ duration: 0.3 }`

4. **Content animation** (use interactions.modalContent):
   - className: `fixed left-1/2 top-1/2 z-[40] -translate-x-1/2 -translate-y-1/2 rounded-glass-xl max-h-[85vh] overflow-hidden flex flex-col`
   - style: `{ ...glassMaterials.liquidModal, width: sizeMap[size] }`
   - Apply top accent: thin gradient bar via ::before or an inner div
   - initial: `{ opacity: 0, scale: 0.88, y: 30, filter: 'blur(8px)' }`
   - animate: `{ opacity: 1, scale: 1, y: 0, filter: 'blur(0px)' }`
   - exit: `{ opacity: 0, scale: 0.95, y: -10, filter: 'blur(4px)' }`
   - transition: `springs.gentle`

5. **Sub-components**:
   - `ModalHeader`: padding `22px 24px 16px`, borderBottom `1px solid rgba(255,255,255,0.15)`. Includes title (18px, weight 700), optional description, and close button (X icon, top-right).
   - `ModalBody`: padding `20px 24px`, `overflow-y-auto`, maxHeight `calc(80vh - 140px)`
   - `ModalFooter`: padding `14px 24px`, borderTop `1px solid rgba(255,255,255,0.15)`, flex justify-end gap-2.5

6. **Close button**: Dialog.Close wrapping a button with X icon from lucide-react. Positioned absolute top-4 right-4.

7. After implementing, add a comment at top: `// IMPORTANT: forceMount on Portal, Overlay, Content prevents Radix pointer-events bug. Do not remove.`

<task type="auto">
  <name>Task 2: Build Toast system with context provider, frosted glass, and slide spring</name>
  <files>src/components/ui/Toast.tsx</files>
  <action>
Create a Toast system with context-based state management (not a third-party toast library).

1. **Types**:
   ```typescript
   type ToastVariant = 'success' | 'error' | 'warning' | 'info';
   interface ToastItem { id: string; variant: ToastVariant; title: string; description?: string; }
   interface ToastContextValue { toast: (variant: ToastVariant, title: string, description?: string) => void; }
   ```

2. **ToastContext + useToast hook**:
   - Create context with `createContext<ToastContextValue | null>(null)`
   - `useToast()`: throws error if used outside provider

3. **ToastProvider**:
   - State: `toasts: ToastItem[]`
   - `addToast`: generates id via `crypto.randomUUID()`, appends to state, schedules removal after 5000ms via setTimeout
   - Max 5 toasts (oldest removed when exceeding)
   - Renders children + toast container

4. **Toast container**:
   - Position: `fixed bottom-6 right-6 z-[60] flex flex-col gap-3`
   - Wrap toasts in `<AnimatePresence mode="popLayout">`

5. **Individual toast** (motion.div):
   - Glass styling: Apply frosted glass base, THEN override backdropFilter to enhanced value: `blur(30px) saturate(200%) brightness(1.05)`. Also add `borderTop: '1px solid rgba(255,255,255,0.5)'`.
   - Classes: `max-w-[400px] rounded-glass p-4`
   - Shadow: `shadow-elevation-16`
   - Border-left per variant (from design.json):
     success: `3px solid #2EAD6B`
     error: `3px solid #E74C5F`
     warning: `3px solid #E8913A`
     info: `3px solid #2B8AFF`

   - Motion animation (interactions.toastSlide):
     initial: `{ opacity: 0, x: 80, scale: 0.9, filter: 'blur(4px)' }`
     animate: `{ opacity: 1, x: 0, scale: 1, filter: 'blur(0px)' }`
     exit: `{ opacity: 0, x: 80, scale: 0.9 }`
     transition: `springs.bouncy`
   - Add `layout` prop for smooth reflow when toasts are added/removed

6. **Toast content layout**:
   - Icon (left): Variant icon from lucide-react (CheckCircle, XCircle, AlertTriangle, Info) with variant color
   - Text (right of icon): title (text-sm font-semibold text-neutral-800) + optional description (text-xs text-neutral-500 mt-0.5)
   - Close button (far right): X icon, ghost style, onClick removes toast immediately

7. **Dismiss on click**: Clicking the close X removes the toast from state immediately (clear its timeout too to prevent double-removal).

Export `ToastProvider` and `useToast`.
