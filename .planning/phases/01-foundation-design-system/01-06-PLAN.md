<task type="auto">
  <name>Task 1: Build Select with Radix primitive and CSS data-state animations</name>
  <files>src/components/ui/Select.tsx</files>
  <action>
Create Select using Radix Select primitive. CRITICAL: Do NOT use AnimatePresence or Motion for the dropdown animation. Radix Select does not support forceMount on Content with position="popper". Use CSS data-state animations instead.

1. **Imports**: `import { Select as RadixSelect } from 'radix-ui'`, cn from cn.ts, glassMaterials from glass.ts, ChevronDown from lucide-react.

2. **Select component** (compound):
   Props:
   ```typescript
   interface SelectProps {
     value?: string;
     onValueChange?: (value: string) => void;
     placeholder?: string;
     label?: string;
     error?: string;
     disabled?: boolean;
     options: { value: string; label: string }[];
     className?: string;
   }
   ```

3. **Trigger** (RadixSelect.Trigger):
   - Match Input glass styling exactly: bg rgba(255,255,255,0.7), backdropFilter blur(8px), border 1px solid rgba(228,230,242,0.8)
   - Classes: `h-10 px-3.5 rounded-clay text-sm text-neutral-700 w-full flex items-center justify-between`
   - Focus shadow: focus-teal from design.json
   - ChevronDown icon on right, text-neutral-400
   - Error: same as Input error state

4. **Content** (RadixSelect.Content):
   - `position="popper"` for dropdown positioning
   - `sideOffset={4}`
   - Apply frosted glass material inline: glassMaterials.frosted
   - Classes: `rounded-clay overflow-hidden shadow-elevation-16 z-[10]`
   - CSS ANIMATION via className:
     Add a CSS class that uses `data-[state=open]` and `data-[state=closed]` for animations:
     ```
     data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=open]:slide-in-from-top-2
     data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=closed]:slide-out-to-top-2
     ```
     OR define inline CSS animation classes via a `<style>` tag or add the dropdownIn/dropdownOut keyframes to a global CSS file or as Tailwind animation utilities.
     Preferred: Add `dropdownIn` and `dropdownOut` to globals.css:
     ```css
     .SelectContent[data-state="open"] { animation: dropdownIn 200ms ease-out; }
     .SelectContent[data-state="closed"] { animation: dropdownOut 150ms ease-in; }
     @keyframes dropdownIn { from { opacity: 0; transform: scale(0.92) translateY(-8px); filter: blur(4px); } to { opacity: 1; transform: scale(1) translateY(0); filter: blur(0px); } }
     @keyframes dropdownOut { from { opacity: 1; transform: scale(1) translateY(0); } to { opacity: 0; transform: scale(0.95) translateY(-4px); filter: blur(2px); } }
     ```
     Create `src/app/globals.css` if it does not exist (or add to existing).

5. **SelectItem** (RadixSelect.Item):
   - Classes: `px-3 py-2 text-sm text-neutral-700 cursor-pointer rounded-md mx-1 outline-none`
   - Hover: bg `rgba(59,191,173,0.06)` (via `data-[highlighted]` attribute)
   - Selected: bg `rgba(59,191,173,0.1)` (via data-[state=checked])
   - Include RadixSelect.ItemIndicator with a Check icon

6. **Label and error**: Same pattern as Input.tsx -- label above (12.5px, 500 weight, #2D2F4D), error below (11px, #CC2030, microBounce).

7. Wrap in `RadixSelect.Root` > `RadixSelect.Trigger` + `RadixSelect.Portal` > `RadixSelect.Content` > `RadixSelect.Viewport` > items.

  <files>src/components/ui/Toggle.tsx, src/components/ui/Checkbox.tsx</files>

**Toggle.tsx:**

1. Props: `checked: boolean`, `onCheckedChange: (checked: boolean) => void`, `disabled?: boolean`, `label?: string`, `className?`

2. **Container**: button element, 44px wide, 24px tall.
   - Off state: bg #D2D5E5
   - On state: bg `linear-gradient(135deg, #3BBFAD, #2A9E8E)` (inline style)
   - Transition: background-color 200ms ease

3. **Thumb**: motion.div inside the button.
   - 18px circle (w-[18px] h-[18px]), bg white, rounded-full
   - Shadow: shadow-elevation-4
   - Position: absolute, centered vertically
   - Off: left 3px
   - On: left 23px (44 - 18 - 3)
   - Use Motion `layout` prop and `transition={springs.bouncy}` for the thumb slide with spring bounce
   - Add `layoutId` is NOT needed here -- use `layout` on the thumb div only

4. **Accessibility**: Use `role="switch"`, `aria-checked={checked}`, `tabIndex={0}`, handle Enter/Space for toggle.

5. **Label**: Optional, render next to toggle with text-sm text-neutral-700.

**Checkbox.tsx:**

1. Props: `checked: boolean`, `onCheckedChange: (checked: boolean) => void`, `disabled?: boolean`, `label?: string`, `className?`

2. **Container**: button element, 20px x 20px, rounded-[6px].
   - Unchecked: bg white, border `1px solid #D2D5E5`
   - Checked: bg `linear-gradient(135deg, #3BBFAD, #2A9E8E)` (inline style), border transparent
   - Focus: shadow focus-teal

3. **Check mark**: motion.svg inside, using a Check path.

   - Only render when checked (with AnimatePresence for exit)
   - Check icon: white color, strokeWidth 3

4. **Label**: Optional, render inline-flex next to checkbox with text-sm text-neutral-700, gap-2.

5. Both components should be controlled (no internal state). The parent manages checked state.
