<task type="auto">
  <name>Task 2: Build ImageUploader with glass dropzone and thumbnails</name>
  <files>src/components/ui/ImageUploader.tsx</files>
  <action>
Create ImageUploader with a glass dropzone area and thumbnail grid. This is a PROTOTYPE -- actual file upload is simulated.

1. **Props**:
   ```typescript
   interface ImageItem {
     id: string;
     url: string;
     alt?: string;
   }
   interface ImageUploaderProps {
     images: ImageItem[];
     onAdd?: (urls: string[]) => void; // simulated -- receives URL strings
     onRemove?: (id: string) => void;
     onReorder?: (images: ImageItem[]) => void;
     maxImages?: number;
     className?: string;
   }
   ```

2. **Dropzone area**:
   - Apply frostedSubtle material: glassMaterials.frostedSubtle
   - Border: `2px dashed #D2D5E5` (default), `2px dashed #3BBFAD` on hover/dragover
   - Height: 120px, rounded-glass
   - Center content: Upload icon (lucide) + text "Arrastra imágenes o haz clic para subir" in text-neutral-400 text-sm
   - On hover: bg `rgba(230,248,245,0.5)` (teal tint)
   - On dragover: scale(1.02) via Motion

3. **Click handling**: Hidden `<input type="file" accept="image/*" multiple>` triggered by dropzone click. Since this is a prototype, the "upload" is simulated -- onAdd receives placeholder Unsplash URLs. Actually, for the prototype:
   - On file select or drop: call `onAdd` with placeholder URLs like `https://images.unsplash.com/photo-placeholder?w=800` or just call onAdd with empty array and let the parent handle mock data.
   - Simpler approach: Just call onAdd with an array containing one generated URL on each click. The parent component will manage the actual mock URLs.

4. **Drag and drop on dropzone**: Handle dragover/dragleave/drop events:
   - `onDragOver`: prevent default, set dragActive state
   - `onDragLeave`: clear dragActive
   - `onDrop`: prevent default, clear dragActive, call onAdd

5. **Thumbnail grid**:
   - Below the dropzone: `grid grid-cols-5 gap-3 mt-3` (or flex-wrap)
   - Each thumbnail: 80px x 80px, rounded-clay (14px), overflow-hidden, relative
   - Image: `<img>` with `object-cover w-full h-full`
   - Remove button: absolute top-1 right-1, small X button with bg-black/50 text-white rounded-full w-5 h-5, hover:bg-red-500
   - Selected state (optional): border `2px solid #3BBFAD`, shadow glow-teal

6. **Drag reorder** (simplified):
   - For the prototype, implement basic drag reorder via HTML5 drag and drop (not a library):
     Set `draggable={true}` on thumbnails
     Handle `onDragStart` (save index), `onDragOver` (prevent default), `onDrop` (swap indices, call onReorder)
   - Add Motion whileDrag: `{ scale: 1.03, zIndex: 10 }` with bouncy spring (from design.json interactions.dragReorder)

7. **Empty state**: When no images, show only the dropzone with slightly larger height and prominent text.
