## Cambios realizados

- **Factor input**: `motion.input` más grande (`text-base font-bold`, borde teal-300), glow ring teal al committear (key-based animation).

- **Botón "Quitar" en CitySlot**: botón rojo con label "Quitar" (antes era un X icon gris casi invisible) para remover hotel de esa opción sin tocarlo del pool.

## Decisiones tomadas

| Decision | Razon |
|----------|-------|
| Pool = catálogo puro, noches solo en CitySlot | Las noches son de la opción (4+3), no del pool. El pool es solo "hoteles disponibles". |
| Noches default en multi-destino = 0 (no `paquete.noches`) | Evita la suma falsa (7+7+7+7=28). El admin completa los valores reales. |
| Noches default paquetes sin parseable = 0 | Obliga al admin a completar en lugar de inventar datos ficticios. |
| `prisma db push` en lugar de `migrate dev` | Drift de migration history entre local y Railway impide `migrate dev`. `db push` es equivalente para cambios aditivos. |
| `HOTELES` como categoría enum para proveedores | Aditivo y non-destructive. Mantiene type-safety en el resto del código. |
| Factor editable en Alojamientos Y Precios | Gerónimo arma opciones en Alojamientos y quiere ver el precio inmediatamente sin saltar de tab. |
| Seed con wipe real (no soft-delete) | Se necesita empezar limpio con datos reales. El `deletedAt` no sirve aquí. |

# Bloque 2 — Modernización UI integral (mismo día, segunda sesión)

> **Objetivo:** Transformar el admin de un panel "decorado" (glassmorphism uniforme) a una herramienta moderna tipo Linear/Vercel/Raycast: glass solo en chrome, canvas de trabajo limpio con hairlines, tablas densas, forms estructurados, skeletons accesibles.

---

## Contexto del problema

El diseño original ("Liquid Horizon") aplicaba glassmorphism a todo de forma uniforme: sidebar, tablas, forms, modales. El resultado:
- Labels oscuros sobre fondos oscuros en modales → ilegibles
- Filas de tabla muy altas (`h-[60px]`) → pocas filas visibles
- 5 patrones distintos de filter bar en los módulos
- 4 sistemas de badges inconsistentes
- `rounded-clay`/`rounded-glass-*` mezclados sin escala
- Sin skeleton loading en ningún módulo
- Cmd+K solo cubría el área de contenido (el sidebar lo tapaba)

## Fase 3 — Fixes y polish

### Fix Skeleton invisible

**Problema:** Color original `rgba(236,237,245,0.7)` era casi indistinguible del fondo blanco.
**Fix:** `#D6D8E3` (neutral-200 sólido). Shimmer wave `#D6D8E3 → #E8EAF0`.

### Fix Cmd+K full screen

**Problema:** `SearchModal` tenía `z-[50]` — el sidebar (`z-[100]`) lo tapaba. El overlay solo cubría el área de contenido.
**Fix:** Bump a `z-[300]` (overlay) y `z-[310]` (modal content).

### Fix Cmd+K desalineado

**Causa raíz:** `className="fixed left-1/2 -translate-x-1/2"` de Tailwind entra en conflicto con el `transform` que Framer Motion aplica para animar `scale/y/filter`. Motion sobrescribe el `transform` CSS y el `translateX(-50%)` desaparece.

**Fix en `SearchModal.tsx`:**
```tsx
// ANTES — translate-x conflicto con Motion
<motion.div className="fixed left-1/2 top-[15%] z-[310] -translate-x-1/2 ..." animate={{ scale: 1 }}>

// DESPUÉS — flex container centra, Motion anima libremente
<div className="fixed inset-0 z-[310] flex items-start justify-center px-4 pt-[15vh] pointer-events-none">
  <motion.div className="pointer-events-auto w-full max-w-[580px] ..." animate={{ scale: 1 }}>
```

## Decisiones arquitectónicas clave

| Decisión | Razón |
|----------|-------|
| Glass solo en chrome (sidebar, topbar, login, dashboard hero) | Canvas de trabajo limpio. Patrón Linear/Vercel: chrome con carácter, tablas legibles. |
| Rollout big-bang después de primitives | Si los primitives están bien, la migración es mecánica. 3 agentes en paralelo. |
| Una sola escala de radius (8/12/20/∞px) | Kill `rounded-clay`, `rounded-glass-sm`, `rounded-glass-lg` mezclados. Consistencia. |
| Densidad 44px row height | 12-14 filas visibles arriba del fold a 1440p. Productividad real para Gerónimo y Lucha. |
| Kill row-level stagger (40ms × 20 filas) | 800ms de jitter en cada paginación. Reemplazado por fade-in-up 120ms en el container. |
| JetBrains Mono solo en IDs/precios/fechas/códigos | DM Sans body en 100% de texto operativo. Mono reservado para datos numéricos. |
| PreciosTab sin feature flag (cálculos no tocados) | Los cálculos son el core del negocio. Cualquier refactor de `calcularVentaOpcion` en sesión aparte con gate manual. |
| CatalogEditor genérico tipo-safe | 5 modales copy-paste → 1 implementación. -900 LOC. Cualquier bug nuevo se arregla en 1 lugar. |
