### Bugs corregidos

#### Select de rol no funcionaba dentro de Modal (`src/components/ui/Select.tsx`)
- **Causa**: El portal de Radix Select tenia `z-[10]`, pero el overlay del Modal tiene `z-[30]`/`z-[40]`.
- **Fix**: Cambiado `z-[10]` â†’ `z-[200]` en el `SelectContent`.
- **Detalle adicional**: actualizado borde de `rgba(17,17,36,0.10)` â†’ `rgba(17,17,36,0.14)` para mejor visibilidad.

#### Toggle thumb se salia del track (`src/components/ui/Toggle.tsx`)
- **Causa**: Framer Motion `layout` prop combinado con `style.left` causaba un glitch visual donde el thumb se desplazaba fuera del contenedor.
- **Fix**: Reemplazado `layout` + cambio de `style.left` por `animate={{ x: checked ? 20 : 0 }}` con `style={{ left: 3, top: 3 }}` fijo. Eliminado el `layout` prop por completo.

#### Todos los modulos redirigian a /dashboard en hard reload (`src/app/(admin)/layout.tsx`)
- **Causa**: El layout de admin usaba `isAuthenticated` (booleano derivado de session) que es `false` durante la hidratacion inicial de NextAuth v5. Esto disparaba `router.push('/login')` inmediatamente.
- **Fix**: Usar `const { status } = useSession()` directamente. Solo redirigir cuando `status === "unauthenticated"`. Bloquear render con loading state hasta que `status === "authenticated"`.

#### ServiceProvider.updateX fallaba silenciosamente (`src/components/providers/ServiceProvider.tsx`)
- **Causa**: Los wrappers de update pasaban la entidad completa al Server Action. Prisma rechazaba campos readonly como `id`, `brandId`, `createdAt`, `updatedAt`, `deletedAt`, y relaciones anidadas.
- **Fix**: Agregado `STRIP_KEYS` Set con los campos a excluir y helper `stripForUpdate<T>()`. Aplicado a los 10 update wrappers del provider.

#### Clone de circuito creaba sub-entities sin await (`src/app/(admin)/circuitos/page.tsx`)
- **Causa**: `forEach` no esperaba las Promises de creacion de dias y precios.
- **Fix**: Reemplazado por `await Promise.all([...dias.map(...), ...precios.map(...)])` con try/catch y toast de error.

#### Filtros de tipo en Traslados no conectados (`src/app/(admin)/traslados/page.tsx`)
- **Causa**: El componente `SearchFilter` recibia `filters={[]}` (array vacio) en lugar de los chips de tipo.
- **Fix**: Agregado `tipoFilter` state, `filteredTraslados` con `useMemo` que incluye filtrado por tipo, y chips REGULAR/PRIVADO conectados al state.

## Decisiones tomadas

| Decision | Razon |
|----------|-------|
| `z-[200]` para portal de Select | Garantiza visibilidad sobre cualquier Modal (z-[30]/z-[40]) y Drawer sin depender de stacking context especifico |
| `animate.x` en Toggle en vez de `layout` | Framer Motion `layout` no se lleva bien con posicionamiento absoluto cuando el padre no tiene `layout` tambien. La animacion directa de x es equivalente y mas predecible |
| Eliminar multi-marca de la UI por completo | El cliente opera una sola marca. La UI de seleccion de marca era ruido puro y ademas causaba el bug de proveedores vacios por localStorage con brandId incorrecto |
| `FIXED_BRAND_ID` + limpiar localStorage | Defensivo: si alguien tiene en localStorage `brand-2` de sesiones anteriores, el useEffect lo limpia al cargar |
| `STRIP_KEYS` en ServiceProvider | Principio de menor privilegio en las mutaciones: pasar solo lo que Prisma necesita actualizar, no la entidad completa |
| Favicon SVG sin caja de fondo | Los favicons cuadrados + gradiente + simbolo centrado son el patron generico de Canva/Shields. Una silueta de avion sin caja transmite identidad de viajes sin verse fabricado |
| Barra de acento en FormSection | Alternativa sutil al color de fondo de seccion. No agrega peso visual pero establece jerarquia clara entre secciones sin necesidad de separadores gruesos |

## Hallazgos durante los tests

- **Proveedores aparecian vacios**: `BrandProvider` leia `localStorage` con un `brandId` de sesiones anteriores (ej. `brand-2`). Como no habia proveedores en `brand-2`, la lista aparecia vacia aunque la DB tenia datos en `brand-1`. Fix: `FIXED_BRAND_ID` + limpieza de localStorage.
- **Navegacion a /perfiles como vendedor** usaba `waitUntil: 'networkidle'` en el test y bloqueaba indefinidamente porque Next.js dev server hace POST retries que mantienen la red activa. Solucionado con `waitUntil: 'load'`.
- **Clone de circuito**: el boton "Clonar" esta dentro de un `DropdownMenu` (aria-label="Mas acciones"), no es un boton directo. El helper de test fue ajustado a `openRowActionMenu` + `clickMenuItem`.
- **Delete de circuito con clone**: al buscar por nombre matcheaba 2 filas (original + "Copia de ..."). El test ejecuta delete dos veces consecutivas para limpiar ambas.
- **Traslados usa InlineEditTable**: el boton de eliminar tiene `aria-label="Eliminar"` directo, distinto al patron de DropdownMenu de otros modulos. Helper `deleteFirstRow` detecta ambos patrones.
